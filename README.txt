PROGETTO ARCHITETTURA SOFTWARE ELARDINO-GUEYE-CUBELLI
--------------------------
ci sono tre package perchè abbiamo cercato di sviluppare tre versioni, dato
che la prima pensata non è andata a buon fine.
-- n.b: nel main di ogni progetto ci sono ulteriori annotazioni --
--------------------------
- package ESAME: il pattern utilizzato è il TEMPLATE METHOD (quello presentato),
  però come vedrà seppur la logica del pattern funziona, non è attuabile per il nostro progetto.

- package esame2: il pattern utilizzato è il COMMAND, però il programma rilancia
  un'eccezione e si ferma. non sappiamo se come logica ci possa stare perchè sembra che complichi
  di più le cose invece che semplificarle.

- package esame_normale: questa è la forma più semplice del progetto senza l'utilizzo di pattern. 
  l'abbiamo inserita principalmente come test e comparatore di complessità per le versioni con i pattern.
